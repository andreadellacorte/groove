# Migration: 0.4.0 → 0.5.0

## Outcome

`.groove/index.md` no longer contains the obsolete `sessions:` key. Session directories exist under the groove memory path, ready for native multi-session tracking.

## Acceptance Criteria

- `sessions:` key is absent from `.groove/index.md` frontmatter
- `<memory>/sessions/`, `<memory>/sessions/specs/`, and `<memory>/sessions/docs/` directories exist
- No other keys in `.groove/index.md` are modified

## Background

0.5.0 replaces the bonfire session backend with native multi-session tracking built directly into groove memory. Sessions are now stored as individual files at `<memory>/sessions/<name>.md`. The `sessions:` config key is no longer read by any groove command.

## Constraints

- This migration is idempotent — safe to re-run if interrupted

### Step 1 — Remove `sessions:` from `.groove/index.md`

If `sessions:` key is present in `.groove/index.md` frontmatter, remove it. If absent, skip.

### Step 2 — Create session directories

Read `memory:` from `.groove/index.md` frontmatter (default: `.groove/memory/`). Create the following directories if they do not exist:

```bash
mkdir -p <memory>/sessions/specs <memory>/sessions/docs
```
