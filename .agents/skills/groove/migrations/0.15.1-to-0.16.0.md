# Migration: 0.15.1 → 0.16.0

## What changed

v0.16.0 introduces:

- `.groove/memory/docs/` directory for reference documentation created by `/groove-work-doc`
- `context-reprime` hook now runs the full prime script instead of echoing a reminder
- Three new bash scripts power the prime output: `groove-utilities-prime.sh`, `groove-utilities-task-prime.sh`, `groove-utilities-memory-prime.sh`

## Outcome

The `docs/` memory directory exists. The `context-reprime` hook in `.claude/settings.json` calls the prime script directly so workflow context is injected automatically at session start.

## Steps

### 1. Create docs directory

```bash
mkdir -p .groove/memory/docs
```

Report: `✓ .groove/memory/docs/ created`

### 2. Update context-reprime hook

If `.claude/settings.json` exists and contains a `context-reprime` or `groove-utilities-prime` entry in `SessionStart`:

- Replace the command value with: `bash .agents/skills/groove-utilities-prime/scripts/groove-utilities-prime.sh`

If `.claude/settings.json` does not exist or has no SessionStart hook: skip — user can run `/groove-admin-claude-hooks` to install.

Report: `✓ context-reprime hook updated` or `ℹ No SessionStart hook found — run /groove-admin-claude-hooks to install`

### 3. Update Cursor context-reprime hook (if applicable)

If `.cursor/hooks.json` exists and contains a `context-reprime` entry:

- Rewrite `.groove/hooks/cursor/context-reprime.sh` to call the prime script and wrap output in `{"additional_context": "..."}`

If `.cursor/hooks.json` does not exist: skip silently.

Report: `✓ Cursor context-reprime hook updated` or `ℹ .cursor/hooks.json not present — skipping`

## Constraints

- Idempotent — `mkdir -p` is safe to re-run; hook replacement checks before modifying
- Do not create `.cursor/` if it does not exist
